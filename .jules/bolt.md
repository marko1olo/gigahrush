## 2023-10-25 - Extracting logic from large monolithic functions
**Learning:** Breaking down very long, complex functions (like 350+ lines) into single-responsibility helper functions greatly improves readability, simplifies testing, and reduces cyclomatic complexity. Careful handling of mutable state, return values, and parameters is crucial during extraction. Inline text replacement scripts (`node patch.cjs`) are a fast and reliable way to refactor large sections of code programmatically.
**Action:** When refactoring large functions, identify distinct "phases" or logical blocks. Extract them into appropriately named non-exported helper functions. Pass required context as arguments and return updated state. Verify extraction iteratively using type checking and targeted unit tests.
