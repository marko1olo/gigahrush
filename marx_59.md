## Текущая задача
**Баланс: Переработка лут-таблиц (контекстный дроп, адекватные шансы).**



### Контекст задачи
Глубокая переработка лут-таблиц — контекстный дроп, адекватные шансы. Принципы: 1) Дроп зависит от КОНТЕКСТА комнаты (`RoomType`): кухня → еда, склад → ресурсы, лаборатория → химия/PSI. 2) Дроп зависит от этажа: Living → бытовуха, Ministry → документы/ключи, Hell → оружие/патроны. 3) Diminishing returns: если комната уже обыскана → шанс найти что-то ценное падает на 80%. 4) Scarcity-driven: патроны редки на первых этажах, обильны на Hell и Базе Ликвидаторов. 5) Никогда не дропать plot-critical предметы рандомом.

### Конкретные файлы и паттерны
- **`src/systems/procedural_loot.ts`**: Универсальная лут-система. Изучите веса, `value` фильтры, фракционные предпочтения.
- **Контекстность**: Добавьте параметр `context: { floorLevel, roomType, danger, faction }` в функцию генерации лута. Пример: в `RoomType.MEDICAL` — больше бинтов/таблеток, меньше оружия. В `FloorLevel.HELL` — больше holy water, меньше еды.
- **`src/data/items.ts`**: Каждый `ItemDef` имеет `spawnRooms` / spawn tags. Используйте их для контекстной фильтрации.
- **Баланс**: Слишком много мусора, мало полезного? Или наоборот? Проведите аудит через `npm run content:audit` и подберите веса.
