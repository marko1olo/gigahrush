🎯 **What:** The \`createMobileControls\` function in \`src/mobile.ts\` was excessively long (over 400 lines) and mixed DOM element creation with complex state management and event binding closures. This PR extracts the initial DOM building block into a separate \`buildMobileUi()\` function that returns an interface (\`MobileUi\`) with strictly typed element references. The state variables and closures have been encapsulated into a new \`MobileControlsImpl\` class that implements the \`MobileControls\` interface.

💡 **Why:** By breaking apart the DOM instantiation from the behavioral bindings, the module becomes easier to read and maintain. Instead of relying on a massive closure, state is now managed via class instance properties. This adheres to good object-oriented principles, reduces the cognitive load required to understand the control bindings, and paves the way for easier testing and further refactoring if needed.

✨ **Result:** The \`src/mobile.ts\` file is now much cleaner and more maintainable. The behavior of the mobile controls overlay remains 100% identical.
