# Сценарист 40: Квестовые описания и награды

## Поверхность

Квестовый текст: `desc`, responses, completion/failure lines, reward descriptions. Он должен говорить игроку, что делать, почему это важно, что он получит и кто пострадает.

## Тон

Четкий, diegetic, без heroic framing. Квест - это поручение, долг, просьба, приказ, сделка или вынужденная вылазка.

## Целевые файлы

- `src/data/plot.ts`
- `src/data/contracts.ts`
- `src/systems/quests.ts`: only if messages are embedded.
- `src/render/quest_ui.ts`: only if labels require text changes.
- `src/gen/*`: registered side quests.

## Что переписать

1. Основной plot chain перевести из "сюжетной миссии" в бытовые поручения с конкретной ценой.
2. `desc` должен быть коротким и понятным на карте/quest UI.
3. Награда должна звучать как вещь и риск: патроны, вода, бинт, пропуск, доверие, долг снят.
4. Failure/expiry lines делать жесткими, но не поэтическими.

## Формула desc

Глагол + объект + место/человек + причина/цена.

Пример: "Отнеси пробу Якову, пока банка не начала греться у лифта."

## Примеры строк

- "Поговори с Барни в оружейной. Нужен ствол, не впечатление."
- "Принеси идол Якову. Он платит за образец, Министерство - за молчание."
- "Убей десять тварей у форпоста. Громный даст патроны, если ты не потратишь все по дороге."
- "Выведи взрослую работницу с этажа 69 до рейда. Долг останется, но человек уйдет."
- "Почини герму школы ОБЖ. Дети знают правило, дверь пока нет."

## Игровая задача

Квестовый текст должен помогать играть и держать тон. Игрок не должен угадывать objective из красивой фразы.

## DoD

- Все changed quest desc checked in UI length.
- Plot and contract text uses concrete verbs.
- No "destiny/hero/savior" framing.
- После data changes `npm run typecheck`; systems/render changes `npm run check`.
