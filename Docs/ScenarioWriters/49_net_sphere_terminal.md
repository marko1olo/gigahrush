# Сценарист 49: НЕТ-Сфера и терминальный текст

## Поверхность

НЕТ-Сфера, НЕТ-ТЕРМИНАЛ ГЕН, облачные сообщения, debug-diegetic интерфейсы, chat/system terminal text. Это современная сетевуха внутри бетонного ада, но без cyberpunk-пафоса.

## Тон

Сухой терминал, районный интернет, аварийный чат, системные строки, пользовательские ники, плохая связь. НЕТ-Сфера должна быть странной, но полезной.

## Целевые файлы

- `src/systems/net_sphere.ts`
- `src/render/net_sphere_ui.ts`
- `src/systems/net_terminal_gen.ts`
- `src/data/net_terminal_gen.ts`
- `src/data/procedural_screens.ts`
- `src/data/screen_signals.ts`
- `README.md` only after shipped behavior changes.

## Что переписать

1. Терминальные строки сделать короткими и функциональными.
2. Ошибки связи должны звучать diegetic: "пакет ушел в соседний этаж", "сервер слышит сирену".
3. Не превращать terminal text в лор-стену. Это interface first.
4. НЕТ-имена и события смерти/самосбора должны быть сухими, неприятными, понятными.

## Словарь

"НЕТ-ГЕН", "сессия", "пакет", "heartbeat", "сирена", "чат", "offline", "маркет", "профиль", "не доставлено".

## Примеры строк

- "НЕТ-СФЕРА: связь есть. Не значит, что этаж тот же."
- "Сообщение не доставлено: маршрут пакета ушел в самосбор."
- "NET-... умер. Последний сигнал: герма закрыта снаружи."
- "Маркет молчит. Цены всё равно растут."
- "НЕТ-ТЕРМИНАЛ ГЕН НЕ ОБНАРУЖЕН. Кусок плоти доступа не дал."

## Игровая задача

Терминальный текст должен помогать пользоваться системой и усиливать ощущение живого общего дома, где сеть тоже ненадежна.

## DoD

- All terminal messages reviewed for length/clipping.
- Нет flashy cyberpunk style.
- После render/UI changes visually check canvas; run `npm run check` if systems touched.
