# Сценарист 42: Обычный самосбор

## Поверхность

Тексты обычного самосбора: warnings, start/end lines, NPC warnings, aftermath, shelter messages, map/log cues. Самосбор - местный выброс, но текст должен быть бытовым: сирена, герма, туман, дверь, щель, укрытие, список.

## Тон

Срочный, конкретный, страшный. Не "мир перестраивается", а "герма закрылась не вокруг вас". Не "бездна", а "уплотнитель пустил туман".

## Целевые файлы

- `src/systems/samosbor.ts`
- `src/data/samosbor_variants.ts`
- `src/data/samosbor_director.ts`
- `src/systems/ai/barks.ts`
- `src/data/context_lines.ts`
- `src/data/rumors.ts`
- `src/data/notes.ts`

## Что переписать

1. Проверить все start/end/warning/aftermath lines на пафос и одинаковость.
2. Разделить каналы: сирена, экран, NPC, карта, log, post-rumor.
3. Предупреждение должно давать решение: к герме, из зоны, не открывать, закрыться раньше, проверить карту.
4. Aftermath должен создавать проблему: дверь не та, контейнер вскрыт, список спорит, монстр остался, вода ушла.

## Словарь

"сирена", "герма", "уплотнитель", "фиолетовый", "туман", "щель", "отбой", "список", "зона", "не открывай".

## Примеры строк

- "САМОСБОР через 30 секунд: зона 12. К герме или за границу зоны."
- "Герма закрылась. Уплотнитель держит, если ты не будешь дергать ручку."
- "Отбой прошёл. Коридор вернулся не весь."
- "Список укрытых не сошелся. Домком уже ищет ручку."
- "Фиолетовый туман ушел в вентиляцию. Это не значит, что он ушел из людей."

## Игровая задача

Текст самосбора должен помогать выжить и оставлять последствия для слухов/квестов.

## DoD

- All samosbor warnings are actionable.
- Aftermath has concrete objects/consequences.
- Нет эпического framing.
- После systems changes `npm run check`.
