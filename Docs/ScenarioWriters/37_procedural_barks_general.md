# Сценарист 37: Процедурные реплики и barks

## Поверхность

Процедурные barks - короткие фразы NPC в мире. Они должны поддерживать атмосферу, но не превращаться в бесконечный одинаковый "мрачный афоризм".

## Тон

Коротко, случайно, конкретно. Bark должен звучать как обрывок реального разговора или реакция на ситуацию. Не как цитата из трейлера.

## Целевые файлы

- `src/systems/ai/barks.ts`
- `src/data/context_lines.ts`
- `src/data/dialogue.ts` generic lines.
- `src/systems/samosbor.ts`: warning barks only if touched.

## Что переписать

1. Просмотреть все большие массивы barks и удалить/заменить строки с абстрактным пафосом.
2. Разнести по голосам: citizen, liquidator, cultist, scientist, wild, occupation.
3. Добавить state-aware микро-реплики: голод, жажда, рана, самосбор warning, theft, production, safe zone.
4. Ограничивать длину. Bark должен помещаться в HUD/log и не занимать сцену.

## Формула строки

1 предмет + 1 состояние + 1 маленький вывод.

Пример: "Кран кашляет ржавым. Сегодня воду кипятим дважды."

## Примеры строк

- "Лифт опять теплый. Я сегодня пешком не герой."
- "Хлеб спрячь. Очередь слышит пакеты."
- "Герма щёлкнула раньше сирены. Плохо."
- "Фильтр сухой, а во рту мокрый бетон."
- "Тихо стало не по расписанию."

## Игровая задача

Barks должны делать мир живым и предупреждать игрока без UI-туториала.

## DoD

- Не меньше 100 проверенных/новых barks суммарно, если берется полный проход.
- Каждая faction/occupation группа имеет свой набор.
- Нет повторяющихся красивых афоризмов.
- После правки `npm run typecheck`; визуальный smoke для clipping если строки длинные.
