# Сценарист 38: Слухи и leads

## Поверхность

Слухи - главный мост между атмосферой и gameplay. Они должны не просто "рассказывать лор", а вести к комнате, предмету, опасности, фракции, контракту или решению.

## Тон

Ненадежный, разговорный, конкретный. Слух может ошибаться в причине, но должен быть полезен как направление или предупреждение.

## Целевые файлы

- `src/data/rumors.ts`
- `src/systems/rumor.ts` only if mechanics need adjustment.
- `src/data/context_lines.ts`: rumor-like contextual lines.
- `src/data/contracts.ts`: related assignment hooks.

## Что переписать

1. Для каждого rumor topic проверить: есть ли reveal/lead/action. Если нет - добавить или переписать в атмосферную bark/notes.
2. Убрать слухи, которые звучат как один narrator.
3. Добавить вариативные голоса: сосед, ликвидатор, клерк, рынок, культ, НИИ, дикий.
4. Слух должен давать выбор: идти/не идти, взять предмет, кому отдать, как обойти.

## Словарь

"говорят", "видели", "не при всех", "за водой", "на Z", "за печатью", "не открывай", "платят", "ищи", "лучше с фильтром".

## Примеры строк

- "Говорят, у прачечной гриб не еда, а очередь. Кто первый срезал, того потом искали в мешке."
- "На рынке белую пробу не берут при лампе. Значит, бери лампу, если идёшь сдавать."
- "У рейса 404 кнопка теплая после отбоя. Лифтеры делают вид, что это нормально."
- "Ведомость Истотита ищут не святые. Ищи того, кто первым спросил про пустые строки."
- "Если на кухне дали черный концентрат без вопроса, вопрос уже вписали позже."

## Игровая задача

Каждый слух должен быть маленьким контрактом с неизвестной оплатой.

## DoD

- 50+ rumors reviewed or rewritten.
- Каждый новый rumor has reveal or lead unless intentionally pure ambient.
- Слухи различимы по источнику.
- После правки `npm run typecheck`.
