# Сценарист 39: Записки, документы и лор-обрывки

## Поверхность

Записки - предметный лор. Они должны быть короткими, грязными и полезными. Не писать большие дневники, если игра показывает одну строку в item data.

## Тон

Обрывок, форма, жалоба, акт, инструкция, записка на двери, полевой приказ, долговая бумага. Больше предметов и процедур, меньше "я чувствую, как бездна...".

## Целевые файлы

- `src/data/notes.ts`
- `src/data/chernobog_docket.ts`
- `src/data/pneumomail.ts`
- `src/data/items.ts`: if note item descriptions need support.
- local docs in `src/gen/*` if content modules include note text.

## Что переписать

1. Разобрать существующие `NOTES`: убрать однотипные афоризмы, оставить/усилить полезные предметные.
2. Добавить категории записок: бытовая, НИИ, Министерство, ликвидаторский акт, рынок, культ, производство, транспорт, VOID.
3. Каждая записка должна иметь автора implied by voice.
4. Добавить новые content notes только если они reachable as item, container, quest or rumor.

## Форматы

- Жалоба: кто, на что, куда подать, почему страшно.
- Акт: номер, предмет, сухая формулировка, плохое последствие.
- Обрывок: одна деталь и незаконченный вывод.
- Инструкция: действие, запрет, причина.

## Примеры строк

- "Акт 18-Б: дверь признана временно отсутствующей. Жильцов просим пользоваться соседней стеной."
- "Не ставить белую пробу у зеркала. Предыдущий лаборант теперь смотрит с обеих сторон."
- "Ложки вернуть до отбоя. После отбоя ложки не принимать, даже если стучат."
- "Долг Нила погашен огнем. Бухгалтерия считает дым неубедительным."
- "Полевой приказ: если лифт пахнет йодом, образец несет не тот, кто его взял."

## Игровая задача

Записки должны делать мир плотнее и давать leads, warnings, consequences.

## DoD

- 100 existing notes reviewed in a full pass, at least 40 rewritten if needed.
- 30+ new reachable notes only if connected to systems/content.
- Нет длинных лор-эссе.
- После правки `npm run typecheck`.
