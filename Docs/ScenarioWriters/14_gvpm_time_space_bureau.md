# Сценарист 14: ГПВМ и пространственно-временные чиновники

## Архетип

ГПВМ - бюро, которое обслуживает время, пространство, вчера, завтра и "вверх". Это не маги. Это ведомство с приборами, допусками, вахтерками и инцидентами, где человек вышел на смену через десять циклов и стал проблемой.

## Тон

Сухой техноканцелярит с бытовыми провалами реальности. Смешно только потому, что страшное оформлено как акт.

## Целевые файлы

- `src/data/rumors.ts`: лифтовые, временные, пространственные слухи.
- `src/data/notes.ts`: акты ГПВМ.
- `src/data/contracts.ts`: inspect/repair/deliver tasks for anomaly floors.
- `src/data/procedural_floors.ts`: только если добавляется data text.
- `src/systems/procedural_anomalies/*`: если есть player-facing messages.

## Что переписать

1. Добавить язык "время как коммунальная услуга": задержка вчера, перерасход завтра, вахтерка ускоренного старения.
2. Избегать sci-fi пафоса. Это ЖЭК для пространства-времени.
3. Слухи должны вести к игровым вещам: лифт, петля, короткий путь, пропуск, опасная клетка.
4. Протоколы ГПВМ должны звучать как инструкции, которые уже опоздали.

## Словарь

"цикл", "завтра", "вчера", "плоскость", "вахтерка", "контур", "метр на метр", "ускорение", "опоздавший", "сверка времени".

## Примеры строк

- "Ваше вчера задержано на третьем окне. Без талона не выдаём."
- "Вахтерка старит левую сторону. Проходить правым плечом, детей не пускать."
- "Если лифт пахнет прошлой сменой, не жмите кнопку дважды."
- "ГПВМ сообщает: верх сегодня сверху. Жалобы на обратное принимаются после стабилизации."
- "Человек вернулся через десять циклов и попросил обед. Комиссия признала его несвоевременным."

## Игровая задача

ГПВМ должен дать процедурным аномалиям свой бюрократический голос и сделать странность пригодной для квестов, слухов и документов.

## DoD

- 20+ строк ГПВМ.
- Минимум 5 строк ведут к лифту/аномалии/маршруту.
- Нет космической оперы.
- После правки `npm run typecheck`; при системных сообщениях - `npm run check`.
