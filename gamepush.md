# GamePush Sandbox Testing & Integration Facts

> Этот документ содержит проверенные факты о поведении песочницы GamePush, полученные опытным путем. Следуйте этим правилам, чтобы не сломать тесты.

## 1. Зависание (Hang) на промисах
Песочница GamePush **намертво зависает**, если мы используем `await gp.player.ready` или `await gp.ready`. 
Если внутри `window.onGPInit` или в цепочке загрузки происходит `await`, который не резолвится, песочница обрывает скрипт. 
**Правило:** Никогда не используйте `await` для свойств готовности GamePush внутри `onGPInit` или где-либо еще, что блокирует запуск тестов.

## 2. Автоматические vs Ручные тесты песочницы
Песочница делится на тесты, которые она может проверить сама (галочка ставится слева), и тесты, которые она просит проверить разработчика вручную (зеленый квадрат с галочкой справа).

**Успешно проходят АВТОМАТИЧЕСКИ (галочки слева):**
- ✅ SDK должен инициализироваться
- ✅ Игра должна вызывать метод GameStart (GameReady)
- ✅ Прогресс должен сохраняться в игрока (Для этого нужно вызывать `gp.player.set('score', ...)` и `gp.player.sync()` строго по клику/взаимодействию!)
- ✅ При запуске игры прогресс должен загружаться корректно (Для этого нужен вызов `gp.player.get('progress')`)

**Тесты для РУЧНОЙ проверки (нужно проставить галочки справа самому):**
Остальные тесты песочница предлагает отметить разработчику вручную, так как не всегда может их однозначно перехватить. Наша игра **полностью удовлетворяет** им всем:
- ⬜️ **Метод GameStart должен вызываться вовремя:** Dual-path: синхронный вызов из `markPlatformReady()` если SDK preloaded, fallback через user gesture. См. секцию 5.
- ⬜️ **Язык (определиться и fallback на англ):** Игра читает `gp.language` в настройках моста и, если язык не из СНГ, корректно ставит английский (`en`).
- ⬜️ **Звук (контроль СДК и кнопка):** Внутриигровое меню паузы имеет физическую кнопку (Домино-интерфейс), которая дергает `togglePlatformAudioMuted`, вызывая `gp.sounds.mute()` и `gp.sounds.unmute()`.

## 3. Слишком частое сохранение (Rate Limit) и Ошибки полей
Если спамить `gp.player.sync({ storage: 'cloud' })` (например, через `setInterval`), GamePush выдает ошибку "СЛИШКОМ часто сохраняте данные", и тесты прогресса могут отвалиться.
Также **нельзя использовать неизвестные поля** вроде `gp.player.set('sandbox_init', '1')`, так как они не зарегистрированы в панели (вызывает ошибку `Field 'sandbox_init' not exists on player model`).
**Правило:** Использовать только базовые поля `score`, `progress` и вызывать `sync()` только один раз или при реальном сохранении в процессе игры.

## 4. Проверка стека вызовов (User Interaction Stack)
Песочница (и сам SDK) строго проверяет стек вызовов JavaScript! Если вызывать методы сохранения, смены языка или звука из `setTimeout` или `setInterval`, песочница может забраковать их с пометкой "Вызвано не пользователем". Все такие вызовы должны происходить из обработчиков реальных событий пользователя (`click`, `pointerdown`, `keydown`).

## 5. Dual-path gameStart (решение проблемы тайминга)

**Проблема:** `gameStart` и `gameplayStart` вызывались одновременно на первый клик. По документации `gameStart` = «игра загружена, меню готово» (раньше), `gameplayStart` = «геймплей начался» (позже). Простой перенос `gameStart` в async `markPlatformReady()` ломает test 3 — sandbox отвергает вызовы из `Promise.then()` как «не вовремя».

**Решение (проверено, работает):**

1. **Синхронный путь** в `markPlatformReady()`: проверяем `portalGlobal().gp` синхронно. Если SDK уже на глобале (sandbox preloads SDK скриптом) — вызываем `gameStart()` сразу, без Promise chain. Это происходит при показе меню, ДО первого клика.

2. **User-gesture fallback** в `fulfillSandboxTests`: если SDK ещё не загружен к моменту `markPlatformReady()`, `gameStart()` вызывается на первый `pointerdown`/`keydown`.

3. **Флаг `gamePushGameStartSent`** гарантирует ровно один вызов — какой путь сработает первым, тот и вызовет.

**Что НЕ работает (проверено):**
- Вызов `gameStart` из `Promise.then()` / async callback → sandbox помечает "не вовремя" → test 3 FAIL.
- Вызов `gameStart` ТОЛЬКО из user gesture → тайминг совпадает с `gameplayStart` → модерация жалуется.

**Правило:** `gameStart` должен вызываться синхронно (не из Promise/setTimeout). Синхронный путь из boot-кода работает, если SDK preloaded.
