## Текущая задача
**Механики: Универсальная разрушаемость декора (столы, бочки, фичи).**



### Контекст задачи
ПОЛНАЯ разрушаемость объектов окружения. ВСЕ Feature-ы разрушаемы ударами, кроме `Feature.LIFT_BUTTON` (plot-critical) и `Feature.SCREEN` (терминалы). Даже кровати, станки и аппараты — ломай всё.

### Конкретные файлы и паттерны
- **`src/core/types.ts`**: `Feature` enum — TABLE, CHAIR, BED, SHELF и т.д. При разрушении: `world.features[idx] = Feature.NONE`.
- **`src/systems/interactions.ts`**: При ударе по ячейке с Feature — уменьшить HP (sparse map `featureHp: Map<number, number>`).
- **Дроп-таблица по Feature type**:
  - `TABLE` → `'wood_plank'` (80%) или `'metal_scrap'` (20%)
  - `CHAIR` → `'wood_plank'` (90%)
  - `BED` → `'cloth_rag'` (60%), `'spring'` (30%)
  - `SHELF` → `'wood_plank'` (50%), случайный предмет из room loot pool (10%)
  - `MACHINE` → `'metal_scrap'` × 2 (70%), `'electronics'` (20%)
  - `APPARATUS` → `'electronics'` (50%), `'wire'` (40%)
  - `STOVE` → `'metal_scrap'` (80%)
  - `SINK` → `'pipe_fragment'` (70%)
  - `TOILET` → `'pipe_fragment'` (60%), splash effect
  - `LAMP` → тьма в комнате + `'glass_shard'` (90%). ОБНОВИТЬ `world.light` через dirty flag!
  - `CANDLE` → ничего, но тьма
  - `DESK` → `'wood_plank'`, иногда документ из комнаты
- **Version bumps**: `world.surfaceVersion++` при удалении фичи.
- **Path blockers**: Если фича была в `World.pathBlockers` — обновить blocker (убрать запись).
- **`src/render/` mesh pass**: Визуальные слоты мешей привязаны к фичам. При удалении фичи — меш тоже пропадает автоматически.
