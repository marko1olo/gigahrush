## Текущая задача
**Процедурные квесты (Межэтажные). Взятие задания на одном этаже, выполнение на другом. Навигация и сохранение.**



### Контекст задачи
Сейчас процедурные квесты (`QuestType` в `src/core/types.ts`: FETCH, KILL, VISIT, TALK) работают только в рамках текущего этажа. Нужно поддержать межэтажные квесты.

### Конкретные файлы и паттерны
- **`src/systems/quests.ts`**: Изучите `checkQuestCompletion()`. Добавьте поле `targetFloorKey?: string` в процедурные квесты, чтобы при его наличии квест проверялся только на целевом этаже.
- **`src/data/contracts.ts`**: Шаблоны `SystemAssignment` уже имеют `routeTarget?: QuestRouteTarget`. Используйте этот паттерн для межэтажных процедурных квестов.
- **`src/systems/procedural_floors.ts`**: Здесь хранится `FloorRunState` с visited route keys. Можно использовать для валидации доступности целевого этажа.
- **`src/render/quest_ui.ts`**: Обновите отображение квеста в логе, чтобы показывать целевой этаж.
- **Save/Load**: Новое поле `targetFloorKey` в квесте должно сериализоваться в `save_payload.ts`. Обновите санитайзер.
