## Текущая задача
**Рендер: Улучшение процедурных спрайтов монстров — детализация, силуэт, читаемость.**



### Контекст задачи
После marx_87 все спрайты работают в 128x128. Это даёт 4x больше пикселей для деталей! Задача — **использовать это пространство**: улучшить художественное качество процедурных спрайтов монстров, добавив детали, которые были невозможны в 64x64. Чёткие силуэты, контрастные детали, субпиксельные переходы, читаемость.

### Конкретные файлы и паттерны
- **`src/entities/*.ts`**: Каждый файл содержит `generateSprite()`. Теперь S=128, есть место для:
  - Чёткие edges (outline 1-2px тёмным вокруг силуэта)
  - Контрастные «глаза» или характерные точки (узнаваемость на расстоянии)
  - Gradient shading для объёма вместо монотонного noise fill
  - Мелкие анатомические детали: зубы, когти, швы, раны
- **Приоритет**: SBORKA, TVAR, ZOMBIE, EYE, SHADOW, NIGHTMARE — те, которых игрок видит чаще всего.
- **`src/render/pixutil.ts`**: Утилиты рисования: `rgba()`, `noise()`, `clamp()`. Можно добавить `outline()` хелпер для автоматического edge detection.
- **Зависимость**: Зависит от marx_87 (S=128). Если 87 ещё не влит — работайте с текущим S, улучшая алгоритмы рисования (они подхватят любой S).
- **Не менять save shape.**
