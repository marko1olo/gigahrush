## Текущая задача
**Боевка: Типы Урона (Бронебойный, Дробь, Энерго, Огонь, Пси).**



### Контекст задачи
Полная система типов урона за ОДНУ задачу: enum → маппинг оружия → слот брони → резисты → калькулятор. Это одна цельная фича, разбивать на 6 задач опасно (merge conflicts в shared files). Задачи marx_51–marx_55 будут перенаправлены на другую работу.

### Конкретные файлы и паттерны

#### 1. Enum (core/types.ts — RED file, минимальное добавление)
```ts
export enum DamageType { KINETIC = 0, BUCKSHOT = 1, ENERGY = 2, FIRE = 3, PSI = 4 }
```
Добавить поле `armorDefId?: string` в интерфейс Entity.

#### 2. Маппинг оружия (data/weapons.ts + data/psi.ts)
Добавить `damageType: DamageType` в WeaponDef. Маппинг: дробовики → BUCKSHOT, плазма → ENERGY, огнемёт → FIRE, PSI → PSI, всё остальное → KINETIC.

#### 3. Броня (data/items.ts)
Добавить `resistances?: Partial<Record<DamageType, number>>` в ItemDef. Создать 3 класса: лёгкая/средняя/тяжёлая + фракционные варианты (ликвидаторская = высокий kinetic, культистская = высокий psi).

#### 4. Экипировка (systems/inventory.ts)
`equipArmor(entity, defId)` / `unequipArmor(entity)`. Предмет перемещается между inventory ↔ slot.

#### 5. Калькулятор урона (systems/combat.ts)
`finalDmg = baseDmg * (100 - resist) / 100`. Lookup: `entity.armorDefId` → `ITEMS[defId].resistances[weapon.damageType] ?? 0`.

#### 6. Save (systems/save_payload.ts)
Добавить `armorDefId` в entity serialization. Sanitizer: `armorDefId ?? undefined`.

#### 7. UI (render/stats_ui.ts)
Слот брони в inventory view. При наведении — показать резисты.
